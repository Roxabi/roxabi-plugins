# Fumadocs Scaffold Reference

This document lists the exact directory layout and files generated by `scaffoldFumadocs()` during `/init` Phase 7b. Use it to verify a completed scaffold or to understand what the function produces before running it.

## Directory Layout

```
{project-root}/
├── apps/
│   └── docs/
│       ├── app/
│       │   ├── layout.tsx
│       │   ├── page.tsx                  # redirects to /docs
│       │   └── docs/
│       │       ├── layout.tsx            # DocsLayout
│       │       └── [[...slug]]/
│       │           └── page.tsx
│       ├── src/lib/source.ts
│       ├── source.config.ts
│       ├── next.config.ts
│       └── package.json
└── docs/
    ├── index.mdx
    └── meta.json
```

## Key Configuration Points

**`source.config.ts`** (inside `apps/docs/`) points its content root to `../../docs`. This means the MDX content directory lives at the project root and is reachable from both the Fumadocs app and non-docs tooling (agents, scripts, CI).

**`src/lib/source.ts`** wires Fumadocs source configuration and exports the `source` object consumed by `[[...slug]]/page.tsx`.

**`app/page.tsx`** is a simple redirect to `/docs` so the root URL forwards to the documentation index without a blank page.

**`app/docs/layout.tsx`** wraps all docs pages in `DocsLayout` from `fumadocs-ui`, providing the sidebar, breadcrumbs, and theme toggle.

## Additive-Only, Skip-and-Warn Rule

`scaffoldFumadocs()` never overwrites existing files. If a target file already exists the function:

1. Skips writing the file.
2. Emits a warning: `[docs-scaffold] skipped {path} — file already exists`.

This rule applies to every file in the layout above. A partial scaffold (e.g. `apps/docs/` already present but `docs/` missing) completes the missing parts without touching what is already there.
