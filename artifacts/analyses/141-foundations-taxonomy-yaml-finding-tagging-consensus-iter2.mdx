## Problem

PR #146 iter-2 review found 7 findings after fix iteration 1. All concern subsumption
enforcement, trust-boundary tightening, and schema documentation. Decisions here land
before Slice 2 (chunking + graduation recall index), making them data-integrity blockers.

## Consensus ρ

**All 7 findings: unanimous Option A (all three agents agree).**
F7: 2-1 Option A (architect + product-lead; security-auditor: advisory, defer if not co-located).

### F1 — `missing-error-handling.description` lists bare-except patterns

**Decision: rewrite description; add forward-ref to `bare-except` (unanimous)**

Remove `except: pass` · `bare except` from the description. Rewrite to cover only:
uncaught coroutine exceptions + ignored `Result`/`Option` return values.
Add: "See `bare-except` for swallowed-exception patterns."

The description is normative — agents read it to decide which class to tag.
Including bare-except patterns creates a two-pass disambiguation requirement
that agents will not reliably perform.

### F2 — SKILL.md Class field rules missing subsumption clause

**Decision: add subsumption bullet to Class field rules (unanimous)**

Bullet text (place after the existing Class field rules):
"Subsumption: `bare-except` subsumes `missing-error-handling` — when both could apply, tag `bare-except` only."

YAML `note` fields are not in scope at agent spawn time; the spawn template is the
SSoT for agent tagging behavior.

### F3 — F2 CI lint deferred without tracking signal

**Decision: create issue + `# SYNC REQUIRED` comment above spawn template (unanimous)**

Comment: `# SYNC REQUIRED: inline class list must match review-classes.yml slugs — see #<N>`
Place immediately above the spawn template code fence.
Mark tracking issue P1 (not P3) — if it slips, graduation drift is silent.

### F4 — `local:<branch-slug>:<sha8>` branch-slug charset unspecified

**Decision: specify slugified format in schema comment (unanimous)**

Branch-slug component: `[a-z0-9-]+`, max 60 chars, lowercased,
non-`[a-z0-9]` characters collapsed to `-`, leading/trailing `-` stripped.

Document in `review-classes.yml` candidate namespace comment adjacent to slug format rule.
An unconstrained branch slug can produce path-traversal or injection in graduation cron
scripts that interpolate the `pr` value.

### F5 — `fix/SKILL.md` Phase 1 no subsumption check

**Decision: add subsumption strip to Phase 1 step 4/5 (unanimous — all three flag as blocking)**

Rule: "∃ `bare-except` ∧ `missing-error-handling` in same finding's `class[]`
  → strip `missing-error-handling`, emit `[subsumption-violation]` log; ¬set C(f) := 0."

This prevents double-counted graduation hits from entering `candidate-classes.jsonl`.
Once Slice 2 builds the recall index, fixing this requires a JSONL migration.
All three panelists recommend treating F5 as blocking (not merely advisory).

### F6 — `bare-except.severity: critical` has no consumer in Slice 1

**Decision: add advisory comment (unanimous)**

Comment on the field: `# advisory only — no consumer reads severity in Slice 1; reserved for Slice 3 dispatch routing`

Keep the field (removal would require a schema re-add in Slice 3 + possible JSONL migration).

### F7 — Candidate slug regex total-length undocumented

**Decision: add length comment (2-1; security-auditor advisory-only if not co-located)**

Comment: `# total max 59 chars (10-char prefix "candidate/" + 1 required-start + up to 48-char suffix)`
