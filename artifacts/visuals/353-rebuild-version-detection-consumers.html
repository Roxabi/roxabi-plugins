Why this matters: the reflex is to stamp a version file into every repo and gate it in CI. But class is not a property of the repo — it is a property of who reads the tag. For the 9 repos with no machine consumer, the version file has no reader, so stamping it only creates a second thing that can disagree with the tag. The correct fix there is version_files: [] and deleting the file: an artifact that does not exist cannot drift.
