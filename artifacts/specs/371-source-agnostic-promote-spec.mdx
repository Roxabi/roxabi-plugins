Expert-reviewed by dev-core architect + devops + product-lead + doc-writer (2026-07-19); their source-proven findings are folded into the W-steps, criteria, and the new §"Out of Scope" and guard affordance below.

The version **derivation is already source-agnostic** post-#353/#370: `price.sh COMPONENT ANCHOR HEAD…` (sole deriver, reads no branch name), `finalize.ts::classifyFinalize` (pure classifier, `isPromote` is a bool, witnesses WARN-only), and `release-consistency.yml`'s push path (floor check). The `staging→main` train lives in three thin **policy** sites: `preflight.sh` (hardcoded `staging`), `release-consistency.yml` **PR path** (`head==staging` early-green gate), `SKILL.md` step 9b (`isPromote` from `head=staging` PR). This spec adds a **trunk** mode alongside staging-train without altering the shared deriver/classifier — a per-repo generated workflow re-runs the *same* derive→classify→act core on `push:main`.

**Notation** (used throughout):
- `M` — the merge commit pushed to `main` (`${{ github.sha }}` on `push:main`).
- `M^1` — first parent of `M` (the `main` tip before the merge).
- `COMPONENT` — `release.component` from stack.yml (tag namespace `<COMPONENT>/v*`).
- `DERIVED` — `price.sh COMPONENT M^1 M` (next version); on `price.sh` exit 10 (first release) → `0.1.0`.
- `BASE` — `price.sh --base-only COMPONENT M^1` (tag floor); `""` on first release.
- `parentCount` — `$(( $(git rev-list --parents -n1 M | wc -w) - 1 ))` (2 for a merge commit).

**Decision references** (from #353 / analysis; inlined so this spec reads standalone):
| Ref | Meaning |

## Goal

A repo sets `release.model: trunk` and gets correct `<component>/vX.Y.Z` tags + GitHub releases automatically on merge-to-main, while every staging-train repo releases with **zero** behavioral change and exactly one automation writes each repo's tags.

## Users

- **Primary:** maintainer of any repo wanting trunk-based flow (roxabi-plugins first — the dogfood).
- **Secondary:** the 10-repo staging-train fleet (`roxabi-factory`, `roxabi-live`, `roxabi-boilerplate`, …) — must keep releasing unchanged via the `staging-train` default; downstream `<component>/vX.Y.Z` tag consumers (`[tool.uv.sources]` pins, `on: push: tags` builds in voiceCLI/llmCLI/imageCLI).

## Expected Behavior

**staging-train (default, unchanged):**
```
feature PR → staging → /promote → PR(head=staging→main) → [PR-path 3-way gate] → merge
   → /promote --finalize → tag <comp>/vX.Y.Z + gh release
```
`/promote`, `preflight.sh`, the PR-path three-way witness gate, and `finalize.ts` stay behaviorally unchanged; existing gate-test fixtures produce identical pass/fail outcomes.

**trunk (`release.model: trunk`, release-please retired, `/promote` guarded off):**
```
feature PR → [PR-path: model==trunk → early-green ALL PRs] → MERGE-COMMIT to main
   ── push:main ──▶ auto-release.yml  (concurrency: group per-ref, queue:max)
        checkout(fetch-depth:0, App token) → fetch --tags → run the SHARED 9b core with M=github.sha, isPromote=true:
          parentCount(M)=2 ∧ DERIVED ≠ BASE → finalize.ts(--is-promote true, witnesses empty)
            → reconcile loop: git push tag <comp>/vX.Y.Z + gh release create (idempotent, per-artifact, D16)
        (push-path D14 floor still guards version_files ≥ tag, post-merge)
```

Edge behaviors (all fixture-verified):
- **1-parent merge** (squash / fast-forward): `finalize` REFUSEs (`parentCount≠2`, distinguished from other refuses by reason) → job **exits non-zero (loud red)**, no tag. Merge-commit is a documented+checked trunk prerequisite.
- **Empty payload** (`DERIVED == BASE`, no conventional-commit delta): short-circuit **before** `finalize` → **green no-op**, no tag.
- **Drift** (`finalize` REFUSEs because an existing tag/release points at ≠ M, D16): **loud red** (NOT greened — distinct from empty).
- **First release** (`price.sh` exit 10): `DERIVED = 0.1.0`, `BASE = ""`, `finalize` called with `|| true` guard.
- **Concurrent merges:** `concurrency: { group: auto-release-${{ github.ref }}, cancel-in-progress: false, queue: max }` — pending runs queue FIFO (do **not** collapse to one); each non-empty merge gets its own release.
- **Partial failure** (tag pushed, `gh release create` fails): the per-artifact reconcile loop retries within the run; if the run still dies, `workflow_dispatch` re-runs it (idempotent — `finalize` sees `tagState=points-at-M, releaseState=absent` → `create-release`).

## Out of Scope

- **Executing roxabi-plugins' `main`-default + `staging` deletion** — separate follow-up, unblocked by this (frame Out-of-Scope). This spec swaps roxabi-plugins' *release automation* (release-please → auto-release) but does NOT flip the default branch. Safe because the trunk guard is **narrowed** (#371 B1): `/promote`'s create-PR path stays open while a `staging` branch exists (`status=trunk_promote_pr`), so a staging→main promotion still produces the 2-parent merge-commit that `auto-release.yml` tags on `push:main`; only `/promote --finalize` is refused under trunk (single writer). roxabi-plugins also has no `release-consistency` ruleset yet (PR-path early-green is inert until one is provisioned).
- **Fleet distribution of `price.sh`/`finalize.ts` to non-plugins consumer repos** — the dogfood runs in roxabi-plugins where those scripts are in-repo; how factory/live/boilerplate obtain the deriver in trunk mode is a rollout concern for their opt-in, not #371.
- **Auto-stamping version files** (bot-commit-back) — D4: deferred; hand-stamp via `check-skill-version` hook.
- **Migrating any staging-train repo to trunk** — opt-in, per-repo, later.
