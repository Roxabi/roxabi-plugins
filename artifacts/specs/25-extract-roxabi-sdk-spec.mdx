## Context

Issue #25 — Phase 5 cleanup from clean architecture spec (#20, ADR-005). Shape A selected: move standalone functions from 3 shim files into their corresponding adapter files, update all import paths, delete shims. Additionally fix cross-plugin sqlite3 boundary violation in `get-invoice-details`.

Key constraint: adapter classes have zero production consumers — all 34 files import standalone functions. This spec preserves the standalone function API; consumers only change their import paths.

## Goal

Eliminate dual import paths by consolidating shim code into adapter files, and fix the cross-plugin boundary violation in `get-invoice-details`.

## Users

- Plugin developers maintaining `dev-core` — no more confusion about which import path to use
- `get-invoice-details` plugin — resilient to vault schema changes via SDK function

## Expected Behavior

After this change:
1. `plugins/dev-core/skills/shared/github.ts`, `config.ts`, and `workspace.ts` no longer exist
2. All consumers import directly from `adapters/github-adapter`, `adapters/config-helpers`, `adapters/workspace-helpers`, `ports/workspace`, or `domain/types`
3. `get-invoice-details/scripts/extract.py` calls `roxabi_sdk.paths.vault_index_entry()` instead of raw `sqlite3`
4. `bun typecheck`, `bun lint`, and `bun test` pass
5. No behavior changes — identical runtime behavior before and after

### Module: get-invoice-details SDK fix

| Affordance | Current location | Target location |
|------------|-----------------|-----------------|
| `vault_index_entry()` | N/A (raw sqlite3 in extract.py) | `roxabi_sdk/paths.py` |

`vault_index_entry(category, type, title, content, metadata)` — inserts into vault.db `entries` table. Returns `True` on success, `False` on failure. Graceful degradation: if vault unhealthy or table missing, returns `False`.

`extract.py` replaces its `index_in_vault()` function body with a call to `vault_index_entry()`.
