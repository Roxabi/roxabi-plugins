## Goal

Turn the compress skill into the v2 enabling architecture: mode dispatch with an anti-phantom gate, multi-target scope resolution across both repo layouts, and token-true metrics backed by an append-only Observation-shaped ledger — so trains B–E land as `references/<mode>.md` files and train C's go/no-go has a measured evidence base.

## Users

- **Primary:** Claude Code operators invoking `/compress` on skill/agent files in marketplace-layout repos (name resolution currently broken there) and legacy `.claude/` repos.
- **Secondary:** trains B–E (#310–#313) landing mode bodies into the dispatch slots; the roxabi-cortex PoC consuming the Observation provenance contract exercised by the ledger.

## Decisions

Recorded adaptations and design resolutions (epic #308 adaptation log):

1. **DP drift (standing, commit `1dd4ae8`)**: decision-presentation protocol removed from staging. All decision points use inline `→ present choice **A** | **B**` blocks; plain input requests are direct 1–2-sentence asks; never AskUserQuestion; no reference to `decision-presentation.md`. Issue Mechanism 6 (explicit Read line for the DP reference) is moot; surviving intent: **no undefined shorthand token in the shipped SKILL.md** — every decision-block idiom used is locally self-evident.
2. **`roxabi_sdk.paths` via direct import** (corrected by devops review): `count_tokens.py` inserts the repo/plugin root into `sys.path` (`Path(__file__).resolve().parents[N]`) and does `from roxabi_sdk.paths import get_plugin_data, ensure_dir` — the exact pattern of all five existing SDK-consuming plugins (cv, get-invoice-details, linkedin-apply, image-prompt-generator); the roxabi_sdk cache sync ships the SDK into installed plugin caches (`docs/plugin-cache.md`). Vendored `_lib/paths.py` copies are the retired pre-#25 pattern and must NOT be reintroduced.
3. **No `version` field in plugin.json** (deliberate, epic-wide): CLAUDE.md default policy is SHA-based versioning; adding one would make `check-skill-version.sh` gate every later train (B–E all rewrite compress SKILL.md) and falsify the epic's "#310 only" bump claim.
4. **Trigger timing**: #309 installs ALL composite mode triggers up front ("expand notation" | "lint notation" | "derive pattern from skills"); the anti-phantom gate halts cleanly on not-yet-implemented modes; trains C/D/E description Δ must be ≈0. (#313's "added only in this PR" wording resolved in favor of #309 by the epic run plan; logged for train E's validator.)
5. **Line-budget check shape**: explicit registry constant `SKILL_LINE_BUDGETS = {'compress': 110}` in `tools/validate_plugins.py` (plugin name → max physical lines of its `skills/*/SKILL.md`), not a new plugin.json field — matches the repo's explicit-constant style; no new CI workflow (the script already runs in CI and lefthook). The check function takes injectable `(budgets, root)` params (pattern: `check_class_list_sync`) so the over-budget fixture lives in a pytest temp dir, never committed under `plugins/`; docstring notes the plugin-name keying assumes one budgeted SKILL.md per plugin.
6. **`source_ref` definition**: pre-image content hash of the compressed target — `git hash-object <file>` when available, else SHA-256 hex; **captured at Phase 2 (before any write) and carried into the Phase 5 `append` call** — a post-write hash would be wrong.
7. **Test strategy for `count_tokens.py`**: CI installs neither `tiktoken` nor `anthropic` (ci.yml pip line) — keep it that way. Tier-selection and agreement logic unit-tested via injectable encoder callables; tests needing real tiktoken use `pytest.importorskip('tiktoken')`. `data` block ships as `{"root": "compress"}` only — no `data.files` (nothing to exemplify; avoids `check_examples_exist` tripping on an uncommitted example).

## Expected Behavior

**Dispatch (Phase 0).** `/compress [mode] <scope>` parses the leading token: `compress` (default when omitted) | `derive` | `expand` | `lint` | `glossary`. Ambiguous first token (matches neither a mode nor an existing path/name) → direct ask ("Mode or target? …"). Mode valid ⟺ `references/<mode>.md` exists next to SKILL.md; missing → halt `mode "<m>" not yet implemented` — no improvisation. This PR ships only `references/compress.md` (the compress-mode body extracted from today's SKILL.md), so `derive|expand|lint|glossary` all halt. `argument-hint` mentions neither `--level` nor `--verify`.

**Scope (Phase 1).** Accepts file path | glob | directory | plugin name. Name resolution discovers both layouts: marketplace (`plugins/<p>/skills/*/SKILL.md`, `plugins/<p>/agents/*.md`) and legacy (`.claude/{agents,skills}/`); direct paths keep working as today. In this repo, name `compress` resolves to `plugins/compress/skills/compress/SKILL.md`. Zero matches → halt with the resolution attempts listed. A name matching in both layouts → present choice.

**Read budget.** Scope resolves to N files: N=1 → proceed; N>1 → exactly one batched present-choice (file list + size estimates) before any read beyond discovery; hard cap N≤10 per run, larger scopes chunked into sequential runs; results presented as one consolidated diff with per-file opt-out.

**Token metrics (Phases 2/4/5).** All measurement through `plugins/compress/scripts/count_tokens.py`: per-section token counts before/after, `Δtokens ≈ 0` flagged, char%/line% never presented as savings. Mandatory `method:` ∈ {`anthropic-api`, `tiktoken-proxy`, `estimate`}: API tier when `ANTHROPIC_API_KEY` present; proxy tier computes **both** o200k and cl100k and marks agreement — binary rules (<5% skip, Δ≈0) documented as binding only under `method=anthropic-api`, proxy acts only when both encodings agree; estimate tier labels output with a warning. When API + tiktoken are both available the run emits a one-shot proxy-vs-API calibration line. The L<30 pre-check is re-based on tokens (< ~200 tokens → warn, annotated as a cheap heuristic).

**Ledger.** One row per completed compress run, appended by `count_tokens.py append` with `O_APPEND` to `<vault>/compress/ledger.jsonl` (`get_plugin_data('compress')`, dir `ensure_dir` 0700). SKILL.md contains no Write/Edit path to the ledger. Row shape is literally Observation (ADR-005): `{id: ULID (vendored ~10-line impl), source: 'compress-skill', source_ref: <pre-image hash, Decision 6>, ts: ISO-8601 UTC, category: <mode>, payload_typed: {schema_version: 1, method, target, sections: [{name, tokens_before, tokens_after}], tokens_before, tokens_after, proxy_agreement: bool|null, calibration: str|null, glossary_version: null (reserved — dep #310), level: null (reserved — dep #311)}, correlation: <run-scoped ULID, shared by every row of one multi-file run>}`. Open enum on `category`. Framed in docs as a "κ-shaped calibration ledger exercising the Observation provenance contract" — never claimed to be cortex's κ table.

**Degradation contract.** `Bash` joins `allowed-tools` and is declared in Safety. Bash unavailable → `method: estimate`, `verify: skipped`, **no ledger row**.

**Docs & guards.** README drops "30-60%" for "measured per run". `tools/validate_plugins.py` gains the line-budget check (Decision 5): fails when a budgeted SKILL.md exceeds its cap, passes on the shipped file. Frontmatter `description` net Δ ≤ +15 tokens vs current (measured with `count_tokens.py`, method recorded in the PR); composite triggers carry "notation" lexically; no bare "glossary"/"decompress"; redundant v1 triggers may be pruned to hold the net budget.

## Constraints

- **Marketplace self-containment** — no `~/projects/ssot/*` or machine-local paths in any shipped file; shared refs only via `${CLAUDE_PLUGIN_ROOT}/../shared/`; the plugin must degrade gracefully standalone.
- **Edit repo source only** — never the plugin cache (`~/.claude/plugins/cache/**`).
- **Frontmatter description budget is train-total** — #309 spends it for all of #309–#313 (Decision 4).
- **Ledger is script-only** (O_APPEND) — no Write/Edit path anywhere in SKILL.md; degradation contract when Bash is unavailable.
- **Git hygiene** — no `--force`/`--hard`/`--amend`; stage specific files only.
- **No new CI workflow, no new CI deps** — the line-budget check rides `validate_plugins.py`; tiktoken/anthropic stay out of ci.yml (Decision 7).

## Non-goals

- Mode bodies for `derive`/`expand`/`lint`/`glossary` — trains B–E; here they halt "mode not yet implemented".
- `notation.md` glossary + G1–G4 fidelity guardrails (#310).
- Fresh-agent read-back verification, golden set, `inventory_diff.py` (#311); L0–L3 levels, R13, provenance markers (#311/#312).
- `--level`/`--verify` flags — absent from argument-hint by design.
- Corpus migration / linting of existing files (#312; descope-under-churn memory).
- Aggregate savings claims in README — the ledger will earn those later.

## Edge Cases

| Scenario | Handling |
|----------|----------|
| Ambiguous first token (neither mode nor resolvable target) | Direct ask (1–2 sentences), then dispatch |
| Mode without `references/<mode>.md` | Halt "mode not yet implemented" — never improvise |
| Scope resolves to 0 files | Halt, list attempted resolutions (both layouts) |
| Name matches in both layouts | Present choice between candidates |
| Scope > 10 files | Chunk into sequential ≤10 runs, state chunk plan up front |
| `ANTHROPIC_API_KEY` absent | Proxy tier (o200k + cl100k, agreement flag) |
| `tiktoken` unimportable | `method: estimate` + warning label |
| `tiktoken` imports but `get_encoding()` fails (first-use network fetch, sandboxed env) | broad catch around encoding load → `method: estimate` + warning (pattern: `roxabi_sdk/browser.py` optional-dep sentinel) |
| Proxy encodings disagree | Display both, take no threshold-gated action |
| Bash tool unavailable | `method: estimate`, `verify: skipped`, no ledger row |
| Ledger dir missing | `ensure_dir` (0700) then O_APPEND |
| Target < ~200 tokens | Warn (cheap pre-check heuristic, token-based re-base of v1's L<30) |
| Already-formal target | v1 behavior retained ("tweaks only") until #311 provenance markers land |
| Legacy `.claude/` repo | Name resolution falls back to legacy layout |
