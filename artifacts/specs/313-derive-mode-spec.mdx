3. **SKILL.md 0-delta edit (ONE, precisely scoped)**: line 46 contains TWO sentences — S1 (the live gate rule `Mode valid ⟺ ref(μ) ∃. ∄ → halt: … — ¬improvise a mode body.`) and S2 (the ship/halt enumeration). Replace ONLY S2 with "Today all five mode bodies ship — `references/{compress,glossary,expand,lint,derive}.md`."; **S1 stays byte-verbatim** (it is the anti-phantom guard for unknown modes — `/compress frobnicate` still halts). Verify no other line claims derive halts; wc -l stays 110.
4. **Signature-normalization spec (deterministic, in derive.md verbatim)**:
   - **Extraction unit (the "construct")**: one of — a `Let:`-block binding line; an `I :=`/`V :=` contract line; one table row; one `O_<name>` operation line; a guard-function line; a status-glyph vocabulary line; or a heading plus its first paragraph when the repetition spans whole sections. Segmentation happens BEFORE normalization; each unit yields one signature.
   - **Rules, applied in this order**: (1) NFC-normalize; (2) KEEP structural tokens verbatim — glyphs and glyph SEQUENCES (∀∃∧∨¬→⟺:= etc., kept as-is, never placeheld), structural keywords (`Let:`, the `I :=`/`V :=` binding markers themselves, `O_` prefix, table pipes, heading markup) — the KEEP rule has PRECEDENCE over placeholder rules; (3) placeholder substitution in this intra-order: backtick code spans → `<CODE>` > quoted strings → `<STR>` > paths → `<PATH>` > numbers → `<NUM>` > identifiers → `<VAR>` (an identifier = a contiguous run of word chars incl. underscores/hyphens — compound identifiers like `MAX_FILES` are ONE `<VAR>`; the bound identifier after a kept `I :=` marker is normalized: `I := pipeline_done` → `I := <VAR>`); heading TEXT tokens follow these rules, heading markup is kept; (4) collapse whitespace runs.
   - The normalized string IS the signature key. Two occurrences cluster iff keys are identical OR the LLM judges them semantically equivalent — every semantic-equivalence merge is listed in the report as `judged-equivalent` with both keys shown. (The audit surfaces merges that happened; spuriously-split clusters are the residual risk — contained by report-only + the verifier coverage map.)
5. **MINE caps (named constants in derive.md)**: `MAX_FILES = 40`, `MAX_BYTES = 400_000` — over-cap scope → halt with chunking advice. Per-file Task fan-out: each subagent reads exactly its one file (train-C single-file cap wording) and returns signatures only.
6. **GATE constants**: `MIN_OCCURRENCES = 3`, `MIN_FILES = 3` (labeled placeholders), `FRESHNESS_DAYS = 7` (named placeholder; predicate: `git log -1 --format=%ct -- <file>` older than now − N days ⇒ stable). Amortization: extraction allowed iff co-occurrence-in-one-context-window (stated file chains) OR declared SSoT objective; report shows BOTH repo-static Δ and runtime per-invocation Δ. **Measurement procedure (count_tokens takes file paths only)**: write each fragment (a pointer line, one inline instance, the emitted ref draft) to a scratch temp file → `count_tokens.py count <tmp>` → read `report['tokens_o200k']` under the default `tiktoken-proxy` method (report both encodings if `agreement` is false), or `report['tokens']` under `estimate`/`anthropic-api`; hand-estimates forbidden — every figure in the double-entry is tool-produced.
7. **Report format (pinned, field → source)**: per cluster — `## Cluster <n>: <name>` (name = first ≤5 meaningful tokens of the signature key) · signature key + occurrence table (file:line ×N, from the MINE fan-out results, sorted by (file, line)) · gate results (one line per predicate: `<predicate>: PASS|FAIL — <tool output>`) · route (fold-in <existing target> | new-pattern | dark-matter — schema_fit is an LLM JUDGMENT over the enum {strong, ambiguous, poor} with a one-line justification; no fake numeric thresholds) · emission (R11 draft + Let-bound pointers | R12 principle per D12 | `skipped:` + break-even arithmetic) · double-entry deltas (`repo-static: ±N tokens · runtime/invocation: ±M tokens`, both per D6's procedure) · unified diff (fold-in: diff against the existing target file; new-pattern: the new file as an add-diff). Trailer: dark-matter list, **gate-failed list (its own disposition — instances of clusters that failed a predicate; NOT dumped into dark matter)**, deliberate-loss list (present-choice-accepted), coverage map, ledger row echo. **Coverage map keying**: `instance_id = (file, line, signature-key)`; disposition ∈ {pattern, principle, dark-matter, gate-failed, deliberate-loss}; acceptance identity: |mapped| + |deliberate-loss| = |all mined instances|, zero unmapped.
8. **Verifier invocation (acceptance-split reuse, NOT spawn-template reuse)**: verify.md's compress spawn template is anchor-specific and cannot drive derive verification — derive.md AUTHORS ITS OWN verifier spawn prompt: the report is materialized to ONE scratch file (outside repo/scope — not a report-only violation; the sole persistent write remains the ledger row), the fresh Task reader reads exactly that file (single-file cap wording) and checks: (a) coverage identity per D7 (every instance_id has a disposition; |mapped|+|loss| = |mined|); (b) the deliberate-loss list carries its present-choice acceptance; (c) spot-reproduction: pick one emitted R12 rule + one `ref:` instance, judge whether following the rule reproduces that instance (recorded as a judgment, "not mechanized in this train" disclosed). **verify.md amendment (one line)**: its derive-acceptance line is extended to name the full disposition set (…↦ pattern/principle/dark-matter/gate-failed) — S9 is acceptance-split reuse plus this one-line amendment, stated honestly.
9. **Dogfood run as evidence (slice 2)**: one bounded derive run over `plugins/dev-core/skills/` (≤ MAX_FILES) targeting the three worked examples; the report + verifier output + ledger row are the PoC evidence, recorded in the PR body/run log — the report itself is NOT committed (a live corpus scan rots immediately; house precedent lint.md's counts-rot rule). **Accepted durability trade-off (deliberate-loss idiom)**: PR bodies are editable and CI logs expire — the durable evidence is the vault ledger row, whose `target` field carries `PR #<n>: <scope>` so the chain stays traceable both ways.
10. **Worked examples in derive.md — SIX distinct blocks, ALL AUTHORED IN SLICE 1** (slice 2 validates the three dogfood ones against live reality): ① I/V boilerplate (dogfood, example-only marked); ② pipeline verify-tables (dogfood); ③ status-glyph vocabulary (dogfood); ④ generic non-Roxabi ("N skills sharing an identical retry-policy paragraph"); ⑤ a poor-fit cluster landing in dark matter verbatim; ⑥ a break-even-skip with the full arithmetic shown. Each block: sample signatures, gate outcome, route, emission-or-skip.
11. **cortex_link section**: schema de-risking only; thresholds = placeholders; D15/GEPA/LILO/RC-x credited non-load-bearing.
12. **R12 template embedded verbatim** (genericized; ssot origin cited as analysis evidence only, no path):
    ```
    ## <Principle name> (validated on <instance>: <before → after metric>)
    ∀ <scope>: <rule body>
    trigger: <machine-checkable predicate>
    ref: <source instance 1> · <source instance 2> · …
    gloss: <≤1-line NL name + docstring>
    confidence: <0-100> · ambiguity_flags: [<flags>]
    ```
    Field semantics: `confidence` ∈ [0,100] ∩ ℤ (0 = none, 100 = certain); `ambiguity_flags` controlled vocabulary {conflict, subjective, incomplete, semantic-judgment, schema-fit-boundary}, `[]` allowed.
13. **R11 output check**: before presenting, the mode greps its own draft for `→ *[A-Z]+\([0-9n]+\)`-style bare pointers → any hit = rejected draft, rewrite with the Let-bound form `Q := read ${CLAUDE_PLUGIN_ROOT}/../shared/references/<f>.md — <gloss>`.

## Expected Behavior

**Dispatch.** `/compress derive <glob|dir|plugin>`: explicit mode only (plain `/compress <file>` routes to compress). Anti-phantom gate flips on when derive.md lands.

**MINE.** Scope → caps check (D5) → per-file Task fan-out (signatures only, single-file cap) → normalization per D4 → cluster by identical keys + audited semantic merges.

**GATE.** D6 predicates, all executable; failing clusters listed with which predicate failed (transparency); amortization basis stated per cluster.

**ROUTE.** strong fit → fold-in proposal into the EXISTING shared reference (diff against it; never a duplicate sibling); ambiguous → new-pattern proposal via present-choice; poor → dark matter verbatim (listed, never abstracted).

**CONFLICT.** Candidate vs existing pattern with high overlap + opposed polarity → present-choice showing BOTH texts; no silent-merge path exists in the mode text.

**EMIT.** R11 per D13's pointer contract; R12 per D12's template; break-even per cluster (skip path with arithmetic); report per D7. **Zero writes** to any scoped/repo file; ledger append is the sole write; supersede-not-destroy stated for the v2 apply path.

**Validate.** Train-C verifier with derive acceptance (D8).

## Edge Cases

| Scenario | Handling |
|----------|----------|
| Scope over caps | Halt with chunking advice (named constants cited) |
| Plain `/compress <file>` | Routes to compress mode — never derive |
| Cluster fails one gate predicate | Listed with the failing predicate; not emitted |
| Amortization basis absent (no co-occurrence, no SSoT declaration) | R11 extraction blocked; cluster may still yield R12 or dark matter |
| Poor schema_fit | Dark matter verbatim; report lists it; never forced |
| Conflict detected | Present-choice with both texts; no merge without it |
| Draft contains a bare pointer | Output check rejects; rewrite Let-bound |
| Break-even negative | Skip emission; arithmetic shown |
| Bash unavailable | No ledger row (stated); freshness/counts degrade to "LLM-estimated, human-gated" labels |
| Verifier inconclusive | Report marked "verification inconclusive — human-gated" (train-C rule) |
| gh/git absent for freshness | Freshness marked unverifiable → cluster held (not passed) |
| An earlier train introduced plugin.json.version by merge time | Re-checked immediately before merge (SC16): version field present ⇒ bump it (the gate reads plugin.json); today it is version-less — verified again before label |
