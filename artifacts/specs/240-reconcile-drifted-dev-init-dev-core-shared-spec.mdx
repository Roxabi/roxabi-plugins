## Context

Promoted from: `artifacts/frames/240-reconcile-drifted-dev-init-dev-core-shared-frame.mdx`

#232 established the copy-sync baseline and assigned `leave-alone-pending-reconciliation` to 4 files in `dev-init/skills/shared/` that had drifted from their dev-core counterparts. This spec resolves all 4.

**Concrete diffs (audited pre-spec):**

| File | Delta |
|------|-------|
| `adapters/github-infra.ts` | dev-init missing `import { ConfigError }` + throws raw `new Error(...)` |
| `domain/parse-issue-ref.ts` | dev-init missing `formatRef()` (4-line function + jsdoc) |
| `ports/workspace.ts` | dev-init missing `vercelProjectId?` and `vercelTeamId?` fields |
| `__tests__/domain.test.ts` | dev-init AHEAD â€” has `IssuePort` + `ProjectPort` type checks absent from dev-core (those ports only exist in dev-init) |

## Goal

Bring 3 dev-init files up to dev-core parity and add them to the copy-sync manifest; classify `domain.test.ts` as intentional fork and update ADR 014 accordingly.

## Users

- **Primary:** Roxabi plugin maintainers â€” ensure dev-init ships a correct error contract, complete domain API, and full workspace type
- **Secondary:** dev-init users â€” indirectly benefit from ConfigError fix and formatRef availability

## Expected Behavior

After this change:

1. `bun run sync:shared` copies the 3 reconciled dev-core files to dev-init verbatim (with `@generated` header). The dev-init copies become byte-identical to dev-core.
2. `bun run sync:shared --check` (lefthook `shared-sources-sync` gate) passes on all 3 files.
3. `biome.json` explicitly suppresses lint on the 3 new copy-sync targets (consistent with existing pattern).
4. ADR 014 row for `__tests__/domain.test.ts` reads `intentional-fork` with documented rationale: dev-init has `IssuePort`/`ProjectPort` type-level checks absent from dev-core because those ports only exist in dev-init.
5. ADR 014 rows for the 3 reconciled files read `copy-sync` with `#240` attribution.
6. `bun run typecheck` and `bun run lint` pass across both plugins.
