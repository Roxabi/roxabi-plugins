## Problem

Four files under `dev-init/skills/shared/` have drifted behind or diverged from their counterparts in `dev-core/skills/shared/`. The drift was catalogued during the #232 shared-source governance audit, where each file was assigned `leave-alone-pending-reconciliation` because the divergence wasn't trivially classifiable at that time.

Three of the four divergences are latent bugs or missing exports: `github-infra.ts` throws a raw `Error` instead of the `ConfigError` class (inconsistent error handling surfaced to callers), `parse-issue-ref.ts` is missing the `formatRef()` export (dev-init callers cannot format refs), and `workspace.ts` is missing `vercelProjectId`/`vercelTeamId` fields (Vercel-aware code in dev-init has no workspace contract). The fourth file, `__tests__/domain.test.ts`, has non-trivial divergence whose nature is not yet characterised.

Until reconciled, dev-init ships with a known-weaker error contract, an incomplete domain API, and an incomplete workspace type â€” all of which will only compound as dev-core continues to evolve.
