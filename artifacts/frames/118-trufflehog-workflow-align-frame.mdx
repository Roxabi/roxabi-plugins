## Problem

Lyra PR [#836](https://github.com/Roxabi/lyra/pull/836) split TruffleHog out of `ci.yml` into a standalone `.github/workflows/secret-scan.yml` (job renamed `secrets` ‚Üí `trufflehog`, `cancel-in-progress: false`, independent triggers). Two dev-core skills still target the old in-`ci.yml` layout:

- `plugins/dev-core/skills/ci-setup/cookbooks/scanning.md:13` describes the now-obsolete `secrets` job inside `ci.yml`.
- `plugins/dev-core/skills/shared/adapters/github-infra.ts:41` hardcodes `required_status_checks.contexts: ['ci']` ‚Äî omitting `trufflehog`.

Observable impact: running `/init` on a fresh repo produces a `ci` workflow but no standalone `secret-scan.yml`, and branch protection requires only `ci` ‚Äî not `trufflehog`. Projects that later adopt the standalone workflow (like lyra did) need a manual `gh api PATCH` on every protected branch. `/checkup` does not flag this divergence.

## Who

- **Primary:** dev-core skill authors + any developer running `/init` or `/checkup` on a repo expected to enforce secret scanning.
- **Secondary:** repos already bootstrapped with dev-core (lyra, voiceCLI, imageCLI, etc.) ‚Äî they need convergence on re-run.

## Constraints

- **Idempotency:** `/init` re-run on a repo with `secret-scan.yml` already present must auto-add `trufflehog` to required checks; re-run without it must not.
- **Dynamic detection:** presence of `.github/workflows/secret-scan.yml` probed via GitHub Contents API ‚Äî no hardcoded assumptions.
- **Reference parity:** generated `secret-scan.yml` must match `Roxabi/lyra:.github/workflows/secret-scan.yml` (name `Secret Scan`, job id `trufflehog`, pinned action SHA `v3.94.3`, `--only-verified`, `cancel-in-progress: false`, `push` + `pull_request` + `workflow_dispatch`).
- **Backwards-compat:** existing repos bootstrapped under the old pattern must converge via `/init` or `/checkup` re-run without manual intervention.
- **Unit test coverage:** `buildBranchProtectionPayload` both branches (with + without secret-scan) must be covered.

## Out of Scope

- Changing Lyra's already-shipped `secret-scan.yml` ‚Äî it is the reference, not the target.
- Replacing TruffleHog with another scanner.
- Migrating existing repos' branch protection retroactively outside `/init` / `/checkup` flows.
- Any multi-workflow orchestration beyond splitting TruffleHog out.
