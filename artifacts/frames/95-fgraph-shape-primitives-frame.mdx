## Problem

fgraph has one node shape ‚Äî the rounded rect (`.fgraph-node`) ‚Äî with `.pill` as the only variant. Architecture diagrams communicate through **shape semantics**: a cylinder *is* a database, a hexagon *is* an agent, a diamond *is* a decision. Without these shapes, every node looks the same regardless of its role, forcing readers to rely on text labels alone.

Two external Claude Code skills ([Cocoon-AI/architecture-diagram-generator](https://github.com/Cocoon-AI/architecture-diagram-generator), [yizhiyanhua-ai/fireworks-tech-graph](https://github.com/yizhiyanhua-ai/fireworks-tech-graph)) demonstrate the value: semantic SVG shapes make diagrams self-documenting at a glance.

Shapes are **primitives, not templates** ‚Äî they belong in `fgraph-base.css` as class modifiers, usable from any forge output (chart, guide, epic, gallery headers). Same principle as `.pill`, `.wide`, `.narrow`.

## Who

- **Primary:** Claude (via forge-chart, forge-guide, forge-epic skills) ‚Äî needs shape modifiers to generate semantically meaningful diagrams
- **Secondary:** Mickael ‚Äî reviewing generated diagrams, expects visual distinction between databases, agents, decisions, events, and documents

## Constraints

- Pure CSS only ‚Äî `clip-path` + `border-radius` + pseudo-elements. No SVG `<defs>`/`<use>`, no JS
- Same 0..100 coordinate system ‚Äî no new layout engine
- Shapes must compose with existing tones (`amber`/`cyan`/`purple`/`green`/`red`)
- Shapes must work with existing content (`.fgraph-title`, `.fgraph-sub`, `.fgraph-pill`)
- Hover states and light/dark theme toggle must work on all shapes
- ~60 lines added to `fgraph-base.css` ‚Äî minimal footprint
- Every existing template benefits with zero migration

## Out of Scope

- New layout topologies or templates (shapes enhance existing templates)
- SVG `<defs>` / `<use>` pattern (clip-path is simpler, no ID collisions)
- JavaScript-based shapes or animations beyond CSS `@keyframes`
- PNG/SVG export capabilities
- New coordinate system or layout engine

## Complexity

**Tier: F-lite** ‚Äî single domain (forge CSS primitives), ~4 files touched, known CSS patterns, no architectural unknowns.

- Files: `fgraph-base.css`, `shape-vocabulary.md` (new), `graph-templates/README.md`, `forge-chart/SKILL.md`
- All CSS patterns are well-documented (clip-path, border-radius, pseudo-elements)
- No cross-domain impact ‚Äî shapes extend existing API surface
