## Problem

The forge skill's `base/components.css` lacks the full component vocabulary needed to reproduce `nats-arch-roadmap`-quality output. Currently, diagrams generated by forge skills cannot match the visual richness of hand-crafted architecture diagrams because essential component classes (phase badges, severity cards, connectors, lanes, etc.) are missing.

This blocks the goal of forge being able to generate production-quality architecture diagrams programmatically.

## Who

- **Primary:** Developers using forge skills to generate architecture diagrams via LLM prompts
- **Secondary:** Consumers of forge-generated diagrams (team members, stakeholders viewing docs)

## Constraints

- All components must use semantic color tokens (`--success`, `--warning`, `--error`, `--info`) — aesthetic-agnostic
- Hover transitions: max 0.15s, opacity/color only
- Responsive: collapse gracefully at 768px
- No hardcoded colors — all via CSS variables
- Must render correctly in all 5 forge aesthetics (default, dark, terminal, blueprint, minimal)
- Target file: `base/components.css` (existing file)

## Out of Scope

- JavaScript behavior (CSS-only components)
- New aesthetics
- Changes to existing component classes
- Non-CSS component implementations (SVG-based connectors beyond CSS)

## Components to Add

| Component | Classes | Description |
|-----------|---------|-------------|
| Phase badge | `.phase`, `.phase.m0` … `.phase.m3` | Milestone/status pills, color-coded by phase |
| Severity card | `.card.critical`, `.card.warning`, `.card.info` | Left-border colored cards by severity |
| Connector | `.connector`, `.connector-dashed` | CSS lines between components (1px solid/dashed) |
| Dependency path | `.dep-path`, `.arrow`, `.parallel` | Sequential (→) and parallel (∥) dep chains |
| Lane container | `.lane`, `.lane-header`, `.lane-body` | Parallel work stream visualization |
| Architecture block | `.arch`, `.c-green`, `.c-amber`, `.c-cyan`, `.c-red`, `.c-purple`, `.c-dim`, `.c-bright` | Colored ASCII architecture diagrams |
| Eyebrow header | `.eyebrow` | Uppercase monospace meta label above title |
| Item list | `.item-list`, `.green-arrow`, `.red-arrow`, `.cyan-arrow`, `.amber-arrow` | Lists with color-coded arrow bullets |
| Data table | `.data-table` | Sticky header, row hover, alternating rows, monospace numbers |
| Component box | `.component`, `.component-label`, `.component-value` | Blueprint-style labeled boxes |
